# 第8單元風險評鑑案例分析與實作

## Case Study 1
- 振興經濟委員會(以下簡稱振委會)，因組織變遷，資訊室主任提出要建置新的公文系統、電子郵件系統及員工資料庫，
- 承辦人向神拓電腦公司購買電腦硬體，與相關系統的維護服務，另外還委託神風網訊科技公司，開發一個Web版的電子公文簽核系統［變更管理與委外管理］
- 為了新的公文系統，承辦人採購2台2U主機，一台放置Web前台，一台放置Microsoft SQL Server 2012做為後台資料庫使用，兩台皆安裝Windows Server 2012作業系統，也都安裝防毒軟體，Windows Server與防毒軟體，都設定自動更新［公文主機］
- 由於新公文系統，是振委會運作的`核心系統`，資訊安全長(副主委)非常重視，深怕若有資安事件，會對形象與信譽造成嚴重損害，
- 因此要求不可中斷超過30分鐘，亦要求上班時間，不可發生會對全會同仁，造成重大影響的資安事件［高階長官的要求與期望］
- 規劃初期，承辦人未考量備援環境，機密公文資料僅輸入文號，以密不錄由方式記錄相關簽核資料，公文資料依文書處理手冊規定辦理，
- 為了員工資料庫與電子郵件系統，承辦人另外購買一台2U主機，裝上Linux，使用MySQL資料庫做為員工資料庫，以Linux內建的Send Mail，做為電子郵件使用［員工資料庫與電子郵件］
- 雖然員工資料庫裡，存放員工的出生年月日、國民身分證統一編號、聯絡資訊及撫養資料，
- 但資訊室主任認為，資料庫偶爾才會用到，並不是振委會的關鍵業務，
- 因此對於系統中斷限期，並無特別要求，但提醒承辦人，要注意電腦處理個人資料保護法的相關規定，須妥善保護員工個資，且資料的正確性不得有誤，
- 最後一再強調資訊安全長的要求，不可發生員工資料外洩情形，甚至登上媒體版面，對聲譽造成重大影響［個資議題與高階長官期望］
- 資訊室主任認為，電子郵件是基本的溝通工具，對這個系統沒有任何要求，即使因事件中斷也沒關係
- 承辦人建置完所有設備與系統後，每天都利用神拓電腦提供的備份服務，把資料異地備份到神拓電腦，神拓電腦每月定期提供備份紀錄予承辦人員審查［第三方契約］

- 1.依高階風險評鑑方式，對新公文系統、員工資料庫及電子郵件系統，分別鑑別安全等級程度
- 2.找出有哪些資訊及資通系統`資產`，`資產類別`為何，至少列舉10項
- 3/依找出的資訊及資通系統資產，評定`機密性`、`完整性`及`可用性`，鑑別資訊及資通系統資產`價值`


## Case Study 2
- 新公文系統在上個月剛正式上線啟用，所有公文系統使用者，在上線前1個月，陸續被要求參加新公文系統操作教育訓練，確保使用者可正確使用公文系統，
- 上課方式在大禮堂由講師操作，所有使用者並未實機操作，惟可在內網，下載新公文系統操作手冊與上課教材［教育訓練］
- 新公文系統上線後，近一個月的時間，資安事件通報紀錄發現近50件，
- 歸類問題後，以使用者操作不熟悉居多，但亦因此衍生出，新公文系統後端資料庫資訊不正確問題，須由神風網訊科技維護工程師，進入資料庫做修改，方可解決正確列印統計報表的議題［資安事件］
- 因神風網訊科技，原開發人員多人離職，被指派來排除問題的工程師，為資淺人員且經常換人，1個月發生5次，
- 因修改程式碼後，上線無法正常運作，造成需要由磁帶倒回前版程式碼的情形，往往一次處理，須耗時1~2小時［承包商人員委外管理］
- 新公文系統主機與員工資料庫主機，置於門禁控管電腦機房，每台主機均有雙電源供應器Power Supply，分別接到不同的不斷電系統UPS，每天早晚進機房查看UPS狀態，連續1個月均為正常，1台負載率為65%，另1台負載率為50%［不斷電系統負載］
- 神風網訊科技，為符合振委會緊急需求，在客戶同意下，直接由公司端連線進入客戶端，對員工資料庫系統，做程式修改測試與資料庫資料的修改，防火牆已鎖定僅神風網訊科技，可遠端連線管理［變更管理］
- 經查閱雙方契約，有段內容為「乙方(神風網訊科技)參與甲方系統開發維護案，專案經理如有更換需求，應於1個月前以書面通知甲方(振興經濟委員會)，其他專案人員如需更換，亦應於2週前以書面通知甲方」，
- 因為舊有合作廠商，雙方已有超過5年合作經驗，並未依契約強制要求神風網訊科技，指派資淺工程師或更換人員時，需於契約規定期限前書面通知，
- 主要原因是，考量最熟悉新公文系統的廠商為神風網訊科技，若依契約精神課予罰款，恐無法有效解決頻繁換人的議題［第三方契約］
- 上班時間發生過1次Microsoft SQL 2005資料庫資料損毀，欲倒回備份磁帶資料時，發現早上備份有出現備份錯誤訊息，經判讀須重新備份，但尚未重新備份。
- 此時，拿出前天早上備份磁帶時，又發現磁帶毀損，無法正確讀出，經倒回2天前備份磁帶資料成功後，使用者需補登2天的公文資料，怨聲載道［委外廠商與備份議題］
- 在新公文系統正式上線使用3個月後，神風網訊科技，於甲方同意下，仍負責甲方有新進同仁時開立帳號，與同仁離職時刪除帳號的工作，且可讓神風網訊科技，由公司端連線至甲方，直接開立或刪除帳號，以解決甲方迫切需要［第三方風險］
- 核心交換器的設備維護契約，載明「故障時，經甲方通知，需於1小時內到場，4小時內修復完畢，8小時內無法修復，需以同等級機型替代供甲方使用」［第三方契約］
- 內部稽核時，政風室發現新公文系統，除管理者帳號admin外，尚有admin2與admin3是最大管理權限帳號，惟神風網訊科技與資訊部門，均無法說明admin3帳號使用者為何人，並抽樣發現3個離職同仁帳號，仍存在新公文系統中［帳號註冊與註銷］
- 新公文系統正式上線使用1個月後，同仁通報查詢速度非常慢，或尚未查詢到資料畫面就凍結的事件數，每週平均達15件以上，經查契約，明定開啟公文系統任一查詢功能畫面，反應時間不得超過3秒［容量管理］
- 新公文系統在週六無預警停電4小時，週一上班時，經同仁通報發現，重新啟動新公文系統後，於10分鐘內順利恢復正常運作［資安事件通報］

- 1.鑑別振興經濟委員會，可能無法承受的風險，依風險高低程度，寫出鑑別的資訊及資通系統資產為何，威脅與脆弱性有哪些，至少列舉5項
- 2.挑選2項不可接受的風險，說明採取的控制措施，降低風險到可接受風險
  - 提示：在挑選不可接受風險時，需與中高階長官的期望有關
